# 100Hires Portfolio Project - Step 1

## Project Overview

This repository represents the first step of the 100Hires portfolio project.  
It documents the initial environment setup, tooling preparation, and repository initialization that establish the foundation for future technical deliverables.

## Tools Installed

- Cursor IDE
- Claude Code (extension)
- Codex (extension)

## Steps Completed

1. **Installed Cursor IDE**
   - Downloaded and installed Cursor IDE on Windows.
   - Completed first-launch setup and verified the editor was running correctly.

2. **Configured AI Extensions**
   - Installed **Claude Code** extension in Cursor.
   - Installed **Codex** extension in Cursor.
   - Confirmed both extensions were active and available in the development workflow.

3. **Initialized Repository**
   - Created local project directory for the 100Hires portfolio assessment.
   - Initialized Git version control and connected the local repository to GitHub.
   - Prepared the repository structure for ongoing project commits and documentation.

## Challenges & Solutions

### 1) Extension Authentication and Access
**Challenge:** One or both AI extensions initially required authentication steps that were not obvious in the editor UI.  
**Solution:** I reviewed the extension documentation and followed the official login flow (account sign-in, token/session verification, and extension reload). After re-authentication, both tools were available.

### 2) Git Remote Setup (SSH vs HTTPS)
**Challenge:** Deciding between SSH and HTTPS for GitHub remote access and handling initial authentication prompts.  
**Solution:** I compared both methods via GitHub documentation. I selected the approach best suited to my environment, validated credentials/keys, and confirmed repository access by testing `git remote -v` and push connectivity.

### 3) First-Time Cursor Navigation
**Challenge:** Locating the right panels and commands in Cursor (extensions, terminal, source control) took extra time on first use.  
**Solution:** I used Cursor docs and quick command search to map common actions, then created a simple setup routine for opening folders, managing extensions, and using integrated Git features efficiently.
